Perl Tutorial Part 3:
Regular Expressions


REGEX OPERATORS


OPERATOR MEANING
=~ Contains.
!~ Does not contain.
/some string/ Search for the string inside the slashes.
/some string/i Case insensitive string search.
\ Escapes special characters (.*?+[](){}^$|\).
\Qsome string\E Escape sequence contained within \Q and \E.
^ Match beginning of string or line.
$ Match end of string or line.
. Match any character.
| "Or". Yes|Maybe matches Yes or Maybe.
? Preceding element occurs either 0 or 1 times.
+ Preceding element occurs 1 or more times.
{x,y} Preceding element occcurs between x and y times.
{,y} Preceding element occurs at most y times.
{x,} Preceding element occurs at least x times.
{x} Preceding element occurs exactly x times.

WILDCARD OPERATORS


OPERATOR MEANING
^ Negates a character class.
Ex: [^abc] is not a, b, or c.
[] Range selector.
\d or [0-9] Represents digits 0-9.
\w or [0-9/a-Za-z_] Represents any number or letter.
\s [ \t\n\r] A whitespace character (space, tab, newline, return).
\D or [^0-9] Not a digit.
\W or [^0-9A-Za-z_] Not a digit nor letter.
\S or [^ \t\n\r] Not whitespace.
\b Match a boundary (whitespace or the beginning/end of a line).

POSIX & UNICODE CLASSES


CLASS MEANING
[[:alpha:]] or [a-zA-Z] An alphabetic character.
[[:alnum:]] or [0-9A-Za-z] An alphanumeric character.
[[:digit:]] or \d A digit, 0-9.
[[:lower:]] or [a-z] A lowercase letter.
[[:upper:]] or [A-Z] An uppercase letter.
[[:punct:]] A punctuation character:
[!"#$%&'()*+,-/:;<=>?@\[\\\]^_`{|}~].

LOOPING THROUGH FILES


   while (<>) {statment}

Opens file specified in command line, loops through it and feeds each line to the special variable $_.

BACKREFERENCES


Perl has special variables $1, $2, $3... that hold the elements that are matched by a regular expression.

SUBSTITUTION


Use the "s///" operator:

   s/matchThis/replaceWithThis/

You can use the "g" modifier to make global substitutions: "s///g".

MODIFIERS


MODIFIER MEANING
/i Case insensitive.
/g Globally substitute or match multiple items.
/m Treat a string as multiple lines.
/s Treat a string as a single line.
/x Allow the use of whitespace/comments inside a match.

SPLIT


The split command breaks its input into a set of substrings. If no delimeter is specified, whitespace is used.

   split delimeter, elementToSplit;

JOIN


Opposite of split; rejoins substrings.

   join delimeter, elementToJoin;
